# SPI Master-Slave Communication using Verilog HDL

## Overview

This project implements the **Serial Peripheral Interface (SPI)** protocol using **Verilog HDL**. It consists of SPI Master and SPI Slave modules that communicate through the standard SPI interface. The design demonstrates **full-duplex serial communication**, where the Master transmits data to the Slave while simultaneously receiving data from the Slave. The project is synthesizable, FPGA-compatible, and verified through simulation, making it suitable for learning RTL design and digital communication protocols.

## SPI Interface Signals

| Signal | Description |
|---------|-------------|
| **SCLK** | Serial Clock generated by the Master |
| **MOSI** | Master Out Slave In |
| **MISO** | Master In Slave Out |
| **CS (SS)** | Chip Select (Active Low) |

## Working Principle

1. The Master initiates communication by pulling the **Chip Select (CS)** signal low.
2. The Master generates the **Serial Clock (SCLK)**.
3. Data is transmitted from the Master to the Slave through the **MOSI** line.
4. Simultaneously, the Slave sends data back to the Master through the **MISO** line.
5. After all 8 bits are transferred, the Master releases the **CS** signal, ending the communication.

## Modules

### SPI Master
- Generates SPI clock
- Controls data transmission
- Sends data through MOSI
- Receives data through MISO
- Controls Chip Select signal

### SPI Slave
- Receives SPI clock from Master
- Reads incoming MOSI data
- Sends response through MISO
- Communicates only when CS is active

### Debouncer
- Eliminates button bouncing
- Provides stable input for FPGA push buttons

### Hex Decoder
- Converts binary data into seven-segment display format
- Used for displaying transmitted or received data
